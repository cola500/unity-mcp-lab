## Outside the repo

`~/Development/` contains only this project. Nothing scavenge-able from sibling repos.

## Risk summary

- **In-build use of Asset Store packages is allowed** by the standard Asset Store EULA — the resulting Quest APK shipping with the assets compiled in is fine.
- **Committing the imported asset folders to this public GitHub repo would violate the EULA**, because that distributes the source. So if we import, we gitignore the imported folder (same pattern we used for Photon Voice's Demos/iOS/VisionOS).
- **A fresh clone won't see the imports.** Anyone who clones this repo will need to add the same Asset Store packages to *their* Unity account and import them. That's friction for collaborators but matches how every Asset Store-dependent project works.

## Recommended smallest next visual-polish slice

**Add a starfield skybox** using `Real Stars Skybox Lite`.

Why this first:
- Single asset, ~6 MB, single material to wire.
- The current campfire scene uses a flat dark-navy `RenderSettings.skybox = null`. Adding a star material as the skybox is one line in `NightAtmosphere.cs`.
- Immediate emotional impact: "outdoors at night" instead of "in a black void".
- Zero impact on networking, voice, or controller flow.
- Zero risk of breaking existing materials/lighting because we're only changing `RenderSettings.skybox`.

Slice shape (would be a separate slice, do NOT execute now):
1. User imports `Real Stars Skybox Lite` via Package Manager → My Assets.
2. Gitignore `Assets/RealStarsSkyboxLite/` (or whatever folder it imports into).
3. Update `NightAtmosphere` to optionally load a skybox material from `Resources` or via serialized reference instead of nulling.
4. Test in headset: look up at the sky during the joint-silence scenario.

## Flags

- All recommended packages carry the **standard Unity Asset Store EULA**. They can be used in the built `.apk` but their source content cannot be committed to the public repo. Existing patterns (gitignored Photon Demos/iOS/VisionOS) already handle this shape.
- `Real Stars Skybox Lite` — assumed Lite means free tier from a publisher offering a paid version. If it turns out to be paid-only, drop back to a CC0 starfield (Polyhaven HDRI or similar).
- `music_zapsplat_atmospheric_tension.wav` already in `Assets/Photon/Demos/` is licence-sensitive. Currently safe because gitignored.
